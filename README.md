ЛР 4: Я бы хотел выбрать проект с библиотекой физических величин. Благодаря мощной системе типов с выводом типов OCaml отлично подходит для этой задачи. Язык позволяет статически проверять совместимость единиц измерения на этапе компиляции, полностью исключая ошибки типа "метры + килограммы" без runtime накладных расходов. Алгебраические типы данных естественно моделируют системы единиц, а модульная система обеспечивает чистую архитектуру библиотеки.

Причины выбора OCaml:

**Система типов и безопасность** OCaml имеет одну из самых мощных систем типов среди фп языков. Вывод типов происходит автоматически, что позволяет писать безопасный код без явного указания типов в большинстве случаев. Система типов OCaml ловит множество ошибок на этапе компиляции.

**Производительность** OCaml компилируется в нативный код и показывает отличную производительность - часто сравнимую с C/C++. Присутствует автоматическая сборка мусора с низкими расходами. Плюс отсутствие runtime-приколов делает производительность предсказуемой.

**Мультипарадигменность** Хотя OCaml в первую очередь функциональный язык, он поддерживает императивное и объектно-ориентированное программирование. Поэтому в проде его приятно использовать, по сравнению с тем же haskel.

**Практическое применение** OCaml активно используется в финансовых системах (Jane Street), так как в торговых операциях важны надежность и производительность, в системном программировании (Unikernel MirageOS) и для компиляторов (Meta).
